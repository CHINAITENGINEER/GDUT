## 第一章：系统整体架构

```
用户请求 recommendTasks(userId, limit)
         |
         v
  [画像检查] --- 画像为空 ---> [冷启动] coldStartRecommend()
         |
         v
  [候选拉取] fetchCandidates()     <- 最近300条，排除已抢单
         |
         v
  [批量预加载] batchLoadPublishers() <- 一次IN查询，消除N+1
         |
         v
  [硬过滤] receivable()            <- 过期/金额/距离 三重过滤
         |
         v
  [精排打分] scoreTask()           <- 五维主分 + 三项奖励
         |
         v
  [排序截取] sorted DESC -> limit(K)
         |
         v
  [推荐理由] buildReason()         <- 中文可解释文字
```

### 三条推荐路径

| 路径 | 接口 | 触发条件 | 特点 |
|------|------|----------|------|
| 主推荐 | recommendTasks() | 画像非空 | 五维打分，个性化强 |
| 冷启动 | coldStartRecommend() | 画像为空 | 热门新任务，无需画像 |
| CF融合 | recommendTasksWithCF() | 任意 | 主推荐+协同过滤，多样性强 |

## 第二章：召回层

### 2.1 候选拉取 fetchCandidates()

取最近300条待接单任务，排除用户已抢单的ID（notIn过滤）。
MAX_CANDIDATE_TASKS=300 控制计算量，不全量扫描。

### 2.2 硬过滤 receivable()

```java
private boolean receivable(Task task, User user) {
    if (deadline != null && deadline.isBefore(now)) return false; // 已过期
    if (amt < minAcceptAmount || amt > maxAcceptAmount) return false; // 金额不符
    if (deliveryType == 1 && dist > maxDistanceKm)  return false; // 超距离
    return true;
}
```

硬约束 vs 软约束：打分只是让分低，不会彻底排除。对截止时间、金额区间这类硬需求，必须用硬过滤确保绝对不可能的任务不出现。

### 2.3 日限制 blocked()

今日抢单达上限 或 今日违规达上限 时停止推荐（平台风控）。

## 第三章：精排五维打分

### 3.1 总分公式

score = w_ability*s_ability + w_category*s_category + w_amount*s_amount
      + w_distance*s_distance + w_delivery*s_delivery
      + urgencyBonus + creditBonus + levelBonus

最终 clamp 到 [0,1]，精确到 4 位小数。

### 3.2 维度一：能力标签匹配 ability

Jaccard 相似度：J(A,B) = |A∩B| / |A∪B|

s_ability = 0.20 + 0.80 × J，值域 [0.20, 1.00]

下界设 0.20 而不是 0 的原因：
完全不匹配时（J=0），给 0.20 基础分，避免 ability 权重对最终分完全无贡献。
如果给 0，则「任何标签不匹配的任务」ability 项贡献都是 0，ability 权重就白费了。

HanLP 改良：只保留词性以 n 开头的名词，排除人名（nr），过滤停用词。

停用词过滤效果（「帮我整理一下Excel数据表格」）：
  不过滤：{帮,我,整理,一,下,Excel,数据,表格}  Jaccard = 2/8 = 0.25
  过滤后：{整理,Excel,数据,表格}              Jaccard = 2/5 = 0.40
相似度提升 60%。

### 3.3 维度二：分类偏好匹配 category

| 情况 | 改良前 | 改良后 |
|------|--------|--------|
| 命中偏好分类 | 1.00 | 1.00 |
| 未命中偏好分类 | 0.35 | 0.20 |
| 画像为空 | 0.50 | 0.50 |

权重0.25下命中与未命中总分差距：
  改良前：0.25 × (1.00-0.35) = 0.163
  改良后：0.25 × (1.00-0.20) = 0.200，提升 22.7%

### 3.4 维度三：金额区间匹配 amount

梯形函数：区间内满分 1.0，超出按区间宽度 span 线性衰减到最低 0.10。

```java
BigDecimal span = safeHi.subtract(safeLo);
BigDecimal ref  = span.compareTo(new BigDecimal("10")) > 0 ? span : new BigDecimal("50");
// 偏低：score = 1 - (min - amt) / ref，最低 0.10
// 偏高：score = 1 - (amt - max) / ref，最低 0.10
```

改良关键：用区间宽度而非边界值做分母。
  改良前：min=5，amt=0，decay=(5-0)/5=1，得分=0，只差1元得0分，严重不合理
  改良后：span=195，decay=(5-0)/195=0.026，得分=0.974，符合直觉

### 3.5 维度四：距离匹配 distance

线上任务固定 1.0
线下任务分段：
  dist <= maxDist：score = 1.0 - 0.40 × (dist/maxDist)
                  dist=0→1.0，dist=maxDist→0.60
  dist > maxDist：score = max(0.10, 0.60 - 0.50 × (dist-maxDist)/maxDist)
                  dist=2×maxDist→0.10

改良前 bug：超出 maxDist 后得分卡在 0.80 不再衰减，距离惩罚完全失效。
改良后线性衰减到 0.10，惩罚真实有效。
