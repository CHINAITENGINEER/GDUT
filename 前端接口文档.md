# 校园任务接单平台 — 前端接口文档

> 版本：v1.0 | 日期：2026-03-10 | 适用：Vue 3 前端开发

---

## 全局约定

### 请求基础配置

```typescript
// src/utils/request.ts
baseURL: import.meta.env.VITE_API_BASE_URL  // 开发环境: http://localhost:8080
timeout: 10000
```

### 请求头

```http
Content-Type: application/json
Authorization: Bearer <JWT Token>   // 需要登录的接口必须携带
```

### 统一返回体结构

```typescript
interface R<T> {
  code: number       // 业务状态码
  message: string    // 提示信息
  data: T            // 业务数据
  timestamp: number  // 服务器时间戳（毫秒）
}
```

### 业务状态码说明

| code | 含义 | 前端处理建议 |
|------|------|-------------|
| 200 | 成功 | 正常处理 data |
| 400 | 请求参数错误 | 弹出 message 提示 |
| 401 | 未登录/Token失效 | 清除本地Token，跳转 `/login` |
| 403 | 无权限 | 弹出「无权限」提示 |
| 404 | 资源不存在 | 弹出提示或跳转404页 |
| 409 | 业务冲突 | 弹出 message 提示（如重复抢单） |
| 500 | 服务器错误 | 弹出「服务器异常，请稍后重试」 |

### 时间字段约定

- **后端返回**：所有时间字段均为 **毫秒时间戳（number）**
- **前端展示**：必须调用 `formatTime(ts)` / `formatRelative(ts)` 转换，**禁止直接渲染数字**
- **前端提交**：时间参数传 **毫秒时间戳**，使用 `toTimestamp(date)` 转换

```typescript
import { formatTime, formatRelative, toTimestamp } from '@/utils/format'
```

### ID字段约定

- 所有 ID（用户ID、任务ID等）后端返回为 **字符串（string）**，防止 JS 大数精度丢失
- 前端定义类型时使用 `string`，不使用 `number`

### 金额字段约定

- 所有金额字段为 **number（保留2位小数）**
- 展示时加 `¥` 前缀，使用 `formatMoney(amount)` 工具函数

### 分页请求约定

```typescript
// 所有分页接口请求参数
interface PageQuery {
  page: number      // 页码，从1开始
  pageSize: number  // 每页条数，默认10
}

// 所有分页接口返回结构
interface PageResult<T> {
  records: T[]    // 数据列表
  total: number   // 总条数
  page: number    // 当前页
  pageSize: number
  totalPages: number
}
```

## 一、认证模块（无需Token）

### 1.1 用户注册

```
POST /api/auth/register
```

**请求体**：

```typescript
interface RegisterDTO {
  phone: string       // 手机号（11位）
  studentId?: string  // 学号（选填，绑定校园身份）
  password: string    // 密码（前端MD5加密后传输）
  smsCode: string     // 短信验证码
  nickname: string    // 昵称（2-20字）
}
```

**返回 data**：

```typescript
interface RegisterVO {
  userId: string
  nickname: string
  token: string   // 注册成功自动登录，返回JWT Token
}
```

**规则**：
- 手机号格式：`/^1[3-9]\d{9}$/`，前端校验后再提交
- 密码前端使用 MD5 加密后传输（`import md5 from 'md5'`）
- 注册前必须先调用「发送验证码」接口，验证码有效期5分钟
- 学号非必填，但填写后可使用学号登录

---

### 1.2 用户登录

```
POST /api/auth/login
```

**请求体**：

```typescript
interface LoginDTO {
  account: string   // 手机号或学号
  password?: string // 密码登录（MD5加密）
  smsCode?: string  // 验证码登录（与password二选一）
  loginType: 0 | 1  // 0=密码登录 1=验证码登录
  remember: boolean // 是否记住（影响Token TTL：true=7天，false=1天）
}
```

**返回 data**：

```typescript
interface LoginVO {
  token: string        // JWT Token，存入 localStorage
  userId: string
  nickname: string
  avatar: string
  role: 0 | 1          // 0=普通用户 1=管理员
  level: number        // 接单等级 1-6
  creditScore: number  // 信誉分
}
```

**规则**：
- Token 存入 `localStorage`，key 为 `campus_token`
- 登录成功后根据 `role` 决定跳转：`role=1` 跳管理后台，`role=0` 跳首页
- 401响应时清除本地Token并跳转登录页

### 3.2 批量图片上传

```
POST /api/file/upload/batch
Content-Type: multipart/form-data
```

**请求参数**：`files`（FormData字段名，多文件）

**返回 data**：

```typescript
interface BatchUploadVO {
  urls: string[]  // 图片URL数组，顺序与上传顺序一致
}
```

**规则**：
- 单文件大小 ≤ 5MB，超出提示「单张图片不能超过5MB」
- 支持格式：`jpg` / `jpeg` / `png` / `gif` / `webp`，其他格式拒绝上传
- 单次最多9张，超出提示「最多同时上传9张图片」
- 前端上传前做本地格式和大小校验，减少无效请求

```typescript
// 前端上传示例
const uploadImages = async (files: File[]) => {
  if (files.length > 9) {
    ElMessage.error('最多同时上传9张图片')
    return
  }
  const formData = new FormData()
  files.forEach(f => formData.append('files', f))
  const res = await fileApi.uploadBatch(formData)
  return res.urls  // 得到URL数组后赋值给表单字段
}
```

### 4.4 抢单

```
POST /api/tasks/{id}/grab
```

**请求体**：`{}`

**返回 data**：

```typescript
interface GrabVO {
  lockExpireAt: number  // 锁单过期时间戳，前端展示倒计时
  message: string       // 「抢单成功！请等待发布者确认，剩余20分钟」
}
```

**前置校验规则（后端执行，前端展示对应错误提示）**：

| code | message | 前端提示 |
|------|---------|----------|
| 409 | 今日抢单次数已达上限(5次) | 弹出提示，隐藏抢单按钮 |
| 409 | 今日已抢过该任务 | 弹出提示 |
| 409 | 今日违规次数过多，明日可继续 | 弹出提示，展示违规说明 |
| 409 | 积分不足，无法抢单 | 弹出提示，引导去完成任务获取积分 |
| 409 | 任务已被他人抢单 | 刷新任务状态 |

**规则**：
- 抢单成功后任务状态变为3（已抢单-待协商）
- 前端在任务详情页展示倒计时（`lockExpireAt - Date.now()`），倒计时归零时刷新任务状态
- 抢单冻结1积分，确认后返还，超时/主动取消则扣除

### 4.9 验收任务

```

```

**请求体**：

```typescript
interface TaskVerifyDTO {
  pass: boolean         // true=验收通过 false=不通过
  rejectReason?: string // 不通过时必填，10-200字
}
```

**返回 data**：

```typescript
interface VerifyVO {
  message: string
  // 验收通过时额外返回结算信息
  settlement?: {
    taskAmount: number
    feeDiscount: number   // 接单者等级折扣率
    feeAmount: number     // 实际手续费
    realAmount: number    // 接单者实际到账

}
```

**规则**：
- 只有任务发布者可调用，任务状态必须为5（已完成）
- 验收通过后平台自动结算，计算公式：`实际手续费 = 金额 × 基础费率 × 等级折扣率`
- 验收通过后前端展示结算明细弹窗（含手续费折扣、接单者到账金额）

## 十、接口封装示例（`src/api/task.ts`）

```typescript
import request from '@/utils/request'
import type { TaskQueryDTO, TaskPublishDTO, TaskSubmitDTO, TaskVerifyDTO } from '@/types/task'

export const taskApi = {
  // 任务列表
  list: (params: TaskQueryDTO) => request.get('/api/tasks', { params }),

  // 任务详情
  detail: (id: string) => request.get(`/api/tasks/${id}`),

  // 发布任务
  publish: (data: TaskPublishDTO) => request.post('/api/tasks', data),

  // 抢单
  grab: (id: string) => request.post(`/api/tasks/${id}/grab`),

  // 发布者确认
  confirm: (id: string) => request.post(`/api/tasks/${id}/grab/confirm`),

  // 发布者拒绝
  reject: (id: string) => request.post(`/api/tasks/${id}/grab/reject`),

  // 主动取消抢单
  cancelGrab: (id: string) => request.post(`/api/tasks/${id}/grab/cancel`),

  // 提交成果
  submit: (id: string, data: TaskSubmitDTO) => request.post(`/api/tasks/${id}/submit`, data),

  // 验收
  verify: (id: string, data: TaskVerifyDTO) => request.post(`/api/tasks/${id}/verify`, data),

  // 取消任务
  cancel: (id: string) => request.post(`/api/tasks/${id}/cancel`),

  // 我发布的任务
  myPublished: (params: { page: number; pageSize: number; status?: number }) =>
    request.get('/api/tasks/my/published', { params }),

  // 我抢单的任务
  myGrabbed: (params: { page: number; pageSize: number; status?: number }) =>
    request.get('/api/tasks/my/grabbed', { params }),
}
```
