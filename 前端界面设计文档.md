## 2. 页面设计

### 2.1 登录页 `/login`

**页面结构**：
```
┌─────────────────────────────────────────────────────┐
│                    Logo + 标题                        │
├─────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────┐    │
│  │  Tab: 密码登录 | 验证码登录                   │    │
│  ├─────────────────────────────────────────────┤    │
│  │  账号输入框 (手机号/学号)                     │    │
│  │  密码输入框  [忘记密码?]                      │    │
│  │  验证码输入框 [获取验证码] (验证码登录时显示)   │    │
│  │  ☐ 记住我（7天免登录）                        │    │
│  │  [登录按钮]                                   │    │
│  └─────────────────────────────────────────────┘    │
│                                                     │
│  还没有账号？[立即注册]                              │
└─────────────────────────────────────────────────────┘
```

**组件说明**：
- `ElTabs` 切换登录方式
- `ElForm` + `ElFormItem` 表单校验
- 验证码按钮：发送后60秒倒计时
- 记住我：影响 Token TTL

```typescript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // 需要登录但未登录
  if (to.meta.requiresAuth && !userStore.isLogin) {
    next({ path: '/login', query: { redirect: to.fullPath } })
    return
  }
  
  // 需要管理员权限
  if (to.meta.role === 'admin' && !userStore.isAdmin) {
    next({ path: '/' })
    return
  }
  
  next()
})
```

## 6. 页面流程图

### 6.1 发布者流程

```
登录 → 首页 → 发布任务 → 待支付 → 去支付 → 支付成功
    → 待接单 → 有人抢单 → 确认/拒绝 → 进行中 → 验收
    → 结算 → 互评 → 完成
```

### 6.2 承接者流程

```
登录 → 首页 → 浏览任务 → 抢单 → 等待确认 → 确认后进行中
    → 提交成果 → 等待验收 → 结算 → 互评 → 完成
```

### 6.3 管理员流程

```
登录 → 管理后台 → 用户管理/任务审核/手续费配置/交易统计/争议处理
```

## 7. 关键交互说明

### 7.1 角色切换

- 顶部导航栏显示当前角色
- 点击切换角色时，刷新页面内容
- 角色影响：首页功能入口、我的任务默认Tab

### 7.2 抢单校验提示

| 场景 | 提示方式 |
|------|----------|
| 今日抢单达上限 | 弹窗提示 + 隐藏抢单按钮 |
| 已抢过该任务 | 弹窗提示 |
| 违规次数过多 | 弹窗提示 |
| 积分不足 | 弹窗提示 + 引导完成任务 |

### 7.3 支付流程

- 点击支付 → 模拟成功 → 自动跳转任务详情
- 支付成功后任务状态变为"进行中"

### 7.4 消息提醒

- 未读消息：顶部导航栏显示红点徽标
- 新消息：浏览器标题闪烁提示
