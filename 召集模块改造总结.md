# 召集运动/游戏实时组队模块改造总结

## 一、改造目标
在原有“校园任务接单平台”基础上，新增了一个独立业务模块：

- 发起召集（运动/游戏）
- 召集大厅实时同步
- 参与者点击参加/退出后状态实时同步
- 活动内聊天室消息实时同步
- 发起人结束活动后房间与大厅状态实时同步

整体采用 **REST + WebSocket** 协同实现，前端以 **WebSocket 事件驱动为主，配合 REST 拉取最新状态**，满足“用户侧无手动刷新、无定时轮询”的实时交互要求。

### 2. REST 接口
基础路径：`/api/rally`

- `POST /api/rally`：发起活动
- `GET /api/rally/list`：查询未结束活动列表
- `POST /api/rally/{id}/join`：参加活动
- `POST /api/rally/{id}/quit`：退出活动
- `POST /api/rally/{id}/end`：结束活动（仅发起人）
- `GET /api/rally/{id}/members`：成员列表（仅活动进行中且当前在队成员可访问）
- `GET /api/rally/{id}/history`：聊天历史（仅活动进行中且当前在队成员可访问）

### 3. WebSocket 通道
在 `WebSocketConfig` 中新增：

- 大厅通道：`/ws/rally/lobby`
- 活动房间通道：`/ws/rally/{rallyId}`

说明：
- 使用 token 鉴权（query 参数 `token=...`）
- 活动房间仅允许“当前在队成员”进入
- 大厅连接成功后用于接收活动创建、更新、结束等广播事件
- 房间连接成功后会收到 `joined` 回执事件

### 4. 实时事件
已支持并广播以下事件：

- `rally_created`（新活动创建）
- `rally_updated`（活动人数等状态变化）
- `member_joined`（成员加入）
- `member_quit`（成员退出）
- `message`（聊天消息）
- `rally_ended`（活动结束）
- `error`（错误信息）
- `joined`（房间连接成功回执）

### 5. 数据库改造
修改 `src/main/resources/db/init.sql`，新增三张表：

- `rally_activity`
- `rally_member`
- `rally_message`

并补充了必要索引，例如：
- 活动状态 / 创建时间索引
- 活动成员唯一索引与状态索引
- 消息创建时间索引

### 6. 安全配置
修改 `SecurityConfig` 白名单，放行以下路径：

- `"/rally/**"`（后端静态演示页路径）
- `"/ws/**"`（WebSocket 路径）

> 注：召集模块业务 API 仍按 token 执行鉴权；WebSocket 连接也会在 Handler 中校验 token 有效性。

## 四、联调流程

1. 启动后端（默认 8080）
2. 启动前端（默认 5173）
3. 登录后从首页进入“实时组队”模块
4. 开两个账号（两个浏览器窗口）验证：
   - A 发起活动
   - B 在大厅实时看到活动并参加
   - A / B 在活动房间实时收发消息
   - B 退出后，A 侧成员状态实时更新
   - A 结束活动后，双方实时收到结束状态并退出当前房间视图
