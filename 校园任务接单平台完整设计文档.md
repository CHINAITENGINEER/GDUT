## 1. 项目概述

### 1.1 项目背景

当前校园内存在大量碎片化需求（如代取快递、资料整理、编程协助、代课占位等），需求方与承接方缺乏高效、安全的对接渠道；现有对接方式（微信群、朋友圈）存在信息杂乱、无保障、结算繁琐等问题。本项目旨在搭建一个轻量级、高适配的校园任务接单平台，实现需求发布、抢单、支付、结算的全流程闭环，同时支持平台手续费动态调整和接单等级体系，兼顾实用性与可扩展性。

### 1.2 项目目标

- **功能目标**：实现用户注册登录、任务发布/抢单/验收、支付结算、手续费动态配置、等级体系、双方互评等核心功能
- **技术目标**：采用Spring Boot 3.x + Vue 3主流技术栈，保证系统高可用、响应流畅，突出技术亮点（Redis缓存、权限控制、动态配置）
- **用户目标**：贴合校园用户需求，操作简洁、交互友好
- **比赛目标**：突出校园适配性、技术创新性和业务完整性

### 1.3 团队分工

- 2人团队：Java后端开发 + Vue前端开发

|------|------|

## 3. 核心业务流程（整合版）

```
发布者发布任务 → 任务待支付 → 发布者支付托管 → 任务待接单
        ↓
承接者点击"抢单" → 锁单20分钟（冻结1积分）→ 待协商状态
        ↓
发布者确认接受 → 返还接单者积分 → 任务进行中
        ↓
承接者提交成果 → 发布者验收 → 平台结算（扣除手续费+等级折扣）
        ↓
双方互评 → 任务完成
```

## 4. 功能模块详细设计

### 4.1 用户模块

#### 4.1.1 注册登录
- 支持手机号、学号注册（绑定校园身份）
- 支持密码登录、验证码登录
- 忘记密码：通过手机号验证码重置密码
- 登录状态保持：勾选"记住我"有效期7天

#### 4.1.2 用户信息管理
- 个人资料编辑：昵称、头像、联系方式、个人简介、技能标签
- 角色切换：发布者 ↔ 承接者
- 信誉分：基于互评结果计算，低于60分限制接单
- 安全设置：修改密码、绑定/解绑手机号

#### 4.1.3 等级体系（技术亮点）

| 等级 | 名称 | 升级所需经验 | 手续费折扣 |
|------|------|-------------|-----------|
| Lv.1 | 新手接单者 | 0 | 无折扣 |
| Lv.2 | 初级接单者 | 200 | 九五折 |
| Lv.3 | 熟练接单者 | 1500 | 九折 |
| Lv.4 | 资深接单者 | 4500 | 八五折 |
| Lv.5 | 专业接单者 | 10800 | 八折 |
| Lv.6 | 顶级接单者 | 28800 | 七五折 |

**经验值获取**：
- 完成任务：任务金额 × 2（取整）
- 5星好评：+10 exp
- 4星好评：+5 exp
- 违规记录：-20 exp

### 4.2 任务模块

#### 4.2.1 任务发布
- 填写信息：标题、分类、描述、金额（1-500元）、截止时间、交付方式
- 金额>200元需管理员审核
- 支持上传任务图片（最多9张）

#### 4.2.2 任务浏览与抢单
- 列表筛选：分类、金额、发布时间、关键词搜索
- 抢单前置校验：
  - 单日抢单次数 < 5次
  - 同一任务单日仅可抢1次
  - 违规次数 < 3次
  - 积分 ≥ 1

#### 4.2.3 抢单锁单流程
1. 承接者点击"抢单" → 冻结1积分，锁单20分钟
2. 状态变为"已抢单-待协商"，双方可查看联系方式
3. 发布者操作：
   - 接受：发布者付款托管 → 任务进行中 → 返还接单者积分
   - 拒绝：任务释放 → 待接单 → 返还接单者积分（无违规）
   - 超时：任务释放 → 扣除接单者积分 → 记1次违规

#### 4.2.4 任务验收
- 承接者提交成果（图片/文件）
- 发布者24小时内验收（通过/不通过+原因）
- 争议处理：可申请管理员介入

### 4.3 支付与结算模块

#### 4.3.1 支付
- 发布者支付任务金额至平台托管（模拟支付宝/微信）
- 未支付任务不可被抢单

#### 4.3.2 手续费与结算
- 基础手续费：阶梯收费（默认100元以下5%，100元以上3%）
- 等级折扣：按接单者等级享受手续费折扣
- 结算流程：验收通过 → 扣除手续费 → 余额转入接单者账户 → 计算经验值

### 4.4 评价模块
- 任务结算完成后48小时内互评
- 星级评分（1-5星）+ 文字评价（1-50字）
- 信誉分关联：5星+2分，4星+1星，1星-1分

### 4.5 消息模块
- 系统消息：审核结果、接单提醒、验收提醒、结算通知
- 私信：任务双方沟通

### 4.6 管理员模块
- 用户管理：查看、禁用/启用用户
- 任务管理：审核高金额任务、删除违规任务
- 手续费配置：动态调整（立即生效，刷新Redis缓存）
- 交易管理：查看交易记录、手续费收入统计
- 争议处理：介入调解任务纠纷

## 7. Redis缓存策略

| Key | 内容 | TTL |
|-----|------|-----|
| token:{userId} | JWT Token | 7天 |

## 9. 安全设计

### 9.1 认证
- JWT Token认证
- 密码使用SHA256+盐值加密存储
- Token存入Redis支持主动失效

### 9.2 权限
- Spring Security + @PreAuthorize
- 普通用户：/api/user/**, /api/tasks/**, /api/payment/**, /api/review/**, /api/message/**
- 管理员：/api/admin/**（需role=1）

### 9.3 数据校验
- @Valid + JSR-303注解
- 金额操作使用事务保证一致性

## 10. 任务状态流转

```
待审核(0) ──通过──→ 待支付(1) ──支付──→ 待接单(2)
   │                      │
   └──驳回──→ 已取消(8)    └──取消──→ 已取消(8)

待接单(2) ──抢单──→ 已抢单待协商(3) ──接受+付款──→ 进行中(4)
                              │
                              ├──拒绝──→ 待接单(2) [返还积分]
                              └──超时──→ 待接单(2) [扣积分+违规]

进行中(4) ──提交成果──→ 已完成(5) ──验收通过──→ 已结算(6)
                              │验收不通过
                              ↓
                             争议/进行中
```
