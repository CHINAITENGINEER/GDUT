# 校园任务接单平台需求文档

# 1. 文档概述

## 1.1 文档目的

本文档用于明确校园任务接单平台的核心需求、功能边界、技术要求及业务流程，为项目开发、测试、比赛演示提供统一依据，确保开发过程贴合校园场景、满足比赛评分要点，同时便于2人开发团队明确分工、高效推进。

## 1.2 项目背景

当前校园内存在大量碎片化需求（如代取快递、资料整理、编程协助、代课占位等），需求方与承接方缺乏高效、安全的对接渠道；现有对接方式（微信群、朋友圈）存在信息杂乱、无保障、结算繁琐等问题。本项目旨在搭建一个轻量级、高适配的校园任务接单平台，实现需求发布、接单、支付、结算的全流程闭环，同时支持平台手续费动态调整，兼顾实用性与可扩展性，适配比赛展示需求。

## 1.3 项目目标

- 功能目标：实现用户注册登录、任务发布/接单/验收、支付结算、手续费动态配置、双方互评等核心功能，形成完整业务闭环。

- 技术目标：采用主流技术栈，保证系统高可用、响应流畅，突出技术亮点（如缓存优化、权限控制、动态配置），适配比赛技术评分要求。

- 用户目标：贴合校园用户（学生、教职工）需求，操作简洁、交互友好，解决校园任务对接痛点。

- 比赛目标：突出校园适配性、技术创新性和业务完整性，便于演示展示，提升比赛竞争力。

## 1.4 适用范围

本文档适用于本校园任务接单平台的开发团队（2人团队：Java后端开发、前端开发）、测试验证及比赛展示，明确各环节需求标准，作为开发和验收的核心依据。

# 2. 需求概述

## 2.1 核心业务场景

平台核心围绕“需求方发布任务→承接方接单→完成任务→验收确认→支付结算→双方互评”的流程展开，支持管理员对平台进行管理（用户管理、手续费配置、任务审核等），实现校园任务的高效、安全对接。

## 2.2 用户角色

平台分为3类核心角色，各角色权限清晰、分工明确，适配校园场景及比赛架构展示需求：

- 普通用户（发布者/承接者）：可切换角色，发布任务、浏览接单、完成任务、支付验收、互评等。

- 管理员：负责平台全局管理，包括用户管理、任务审核、手续费动态配置、交易记录查看等。

## 2.3 核心亮点（贴合比赛）

- 校园适配：支持学号登录、校园场景任务分类（代取、代课、编程等），贴合学生核心需求。

- 动态手续费：平台手续费比例可由管理员动态调整，支持按任务金额阶梯收费，体现系统灵活性。

- 全流程闭环：从任务发布到互评完成，实现全流程线上操作，无线下冗余环节，演示效果清晰。

- 技术可展示：包含缓存优化、权限控制、支付模拟等技术点，便于比赛中展示技术实力。

# 3. 详细功能需求

## 3.1 用户模块（核心功能）

### 3.1.1 注册登录

- 支持手机号、学号注册（绑定校园身份，提升安全性，贴合校园场景）。

- 支持密码登录、验证码登录，登录后自动跳转至首页。

- 忘记密码功能：通过手机号验证码重置密码。

- 登录状态保持：勾选“记住我”后，下次打开平台无需重复登录（有效期7天）。

### 3.1.2 用户信息管理

- 个人资料编辑：可修改昵称、头像、联系方式、个人简介（承接者可填写技能标签，便于需求方筛选）。

- 角色切换：普通用户可自由切换“发布者”“承接者”角色，切换后展示对应功能界面。

- 信誉分展示：基于互评结果生成信誉分，信誉分过低（如低于60分）限制接单权限（提升平台可靠性）。

- 安全设置：可修改密码、绑定/解绑手机号，查看登录记录。

## 3.2 任务模块（核心功能）

### 3.2.1 任务发布

- 发布者需填写任务信息：标题、分类（代取快递、资料整理、编程、代课等）、描述、金额、截止时间、交付方式（线上/线下）、是否需要审核。

- 任务金额限制：最低1元，最高500元（贴合校园任务碎片化特点）。

- 任务预览：发布前可预览任务详情，确认无误后提交发布。

- 任务审核：普通任务自动通过，高金额任务（如超过200元）需管理员审核通过后才会展示（降低风险）。

### 3.2.2 任务浏览与接单

- 任务列表展示：按分类、金额、发布时间筛选，支持搜索任务关键词。

- 任务详情：展示任务所有信息、发布者信息、发布者信誉分。

- 接单操作：承接者可点击“接单”，接单前需确认自身技能匹配，接单后任务状态变为“已接单”，不可重复接单。

- 接单限制：信誉分过低、已有未完成任务的承接者，不可接单。

### 3.2.3 任务状态管理

- 任务状态流转：发布中→已接单→已完成→已结算→已互评（状态实时更新，双方可查看）。

- 取消任务：发布者在任务未接单前可取消任务；已接单后，需与承接者协商一致，方可取消。

- 任务提醒：承接者收到接单提醒、任务截止提醒；发布者收到接单提醒、验收提醒（通过平台消息推送）。

### 3.2.4 任务验收

- 承接者完成任务后，提交“完成申请”，上传交付成果（线上交付可上传文件，线下交付可上传凭证照片）。

- 发布者收到验收提醒后，在24小时内完成验收，可选择“验收通过”或“验收不通过”（不通过需填写原因）。

- 验收争议：若双方无法达成一致，可申请管理员介入调解（提升平台公信力，贴合比赛亮点）。

## 3.3 支付与结算模块（核心功能）

### 3.3.1 支付操作

- 发布者发布任务后，需先支付任务金额至平台托管（保障承接者权益），支付方式支持模拟支付宝/微信支付（比赛用沙箱环境）。

- 支付提醒：发布者未支付时，任务处于“待支付”状态，无法被接单，平台持续提醒支付。

- 支付记录：用户可查看所有支付记录（金额、时间、任务关联信息）。

### 3.3.2 手续费结算

- 手续费规则：平台按任务金额的一定比例收取手续费，比例可由管理员动态调整（支持0%-10%可调），支持阶梯收费（如100元以下收5%，100元以上收3%）。

- 结算流程：发布者验收通过后，平台自动扣除手续费，将剩余金额结算至承接者账户。

- 结算记录：承接者可查看结算金额、手续费金额、结算时间。

- 提现功能（可选，比赛加分）：承接者可将账户余额提现至绑定的支付宝/微信账户，提现需审核（管理员审核）。

## 3.4 评价模块

- 互评时机：任务结算完成后，双方可在48小时内对彼此进行评价。

- 评价内容：可填写文字评价（1-50字）、星级评分（1-5星），承接者可评价发布者付款及时性，发布者可评价承接者完成质量。

- 评价展示：评价内容将展示在对方个人主页，供其他用户参考；评价后不可修改、删除。

- 信誉分关联：星级评分与用户信誉分挂钩，5星加2分，4星加1分，3星及以下不加分，1星扣1分。

## 3.5 管理员模块

- 用户管理：查看所有用户信息，可禁用/启用用户账户，查看用户信誉分、交易记录。

- 任务管理：查看所有任务，审核高金额任务，删除违规任务（如虚假任务、违规内容），查看任务状态流转记录。

- 手续费配置：动态调整手续费比例，设置阶梯收费规则，保存后立即生效（无需重启系统，体现技术亮点）。

- 交易管理：查看平台所有交易记录、手续费收入统计，支持按时间、任务类型筛选。

- 系统管理：查看系统日志（登录日志、操作日志），处理用户投诉/争议。

## 3.6 消息模块

- 系统消息：推送任务审核结果、接单提醒、验收提醒、结算提醒、违规通知等。

- 私信消息：支持发布者与承接者之间发送私信，沟通任务细节（如交付方式、修改要求）。

- 消息查看：用户可查看所有消息，标记已读/未读，删除无用消息。

# 4. 非功能需求（比赛重点）

## 4.1 性能需求

- 响应速度：页面加载时间≤1.5秒，接口响应时间≤500ms，支持同时在线1000+用户（贴合校园峰值场景）。

- 缓存优化：热门任务列表、用户信息采用Redis缓存，提升查询效率（比赛技术亮点）。

- 稳定性：系统连续运行无故障时间≥72小时，无卡顿、崩溃现象，适合比赛现场演示。

## 4.2 易用性需求

- 操作简洁：核心操作（发布任务、接单、支付）不超过3步，界面布局清晰，贴合校园用户使用习惯。

- 交互友好：有明确的操作提示（如支付提醒、验收提醒），错误操作有反馈（如接单失败提示原因）。

- 适配性：支持电脑端、手机H5端（可选小程序），响应式布局，适配不同屏幕尺寸。

## 4.3 安全性需求

- 权限控制：基于Spring Security实现角色权限隔离，普通用户无法访问管理员功能，接口需token验证。

- 数据安全：用户密码加密存储（MD5+盐值），支付信息加密传输，防止数据泄露。

- 防违规：禁止发布违规任务（如违法、低俗内容），管理员可快速删除违规内容、禁用违规用户。

## 4.4 可扩展性需求

- 模块可扩展：支持后续新增功能（如任务悬赏、技能认证），无需大幅修改核心代码。

- 配置可扩展：手续费规则、任务分类可灵活配置，适配不同校园场景的需求。

# 5. 业务流程需求

## 5.1 核心业务流程（比赛演示重点）

流程说明：以下流程为平台核心闭环，比赛演示时可按此流程操作，清晰展示功能完整性。

1. 用户注册登录：普通用户通过学号/手机号注册登录，切换至“发布者”角色。

2. 发布任务：填写任务信息，支付任务金额至平台，任务审核通过后展示在任务列表。

3. 接单操作：另一普通用户切换至“承接者”角色，浏览任务、点击接单，任务状态变为“已接单”。

4. 完成与验收：承接者完成任务，提交交付成果；发布者验收通过。

5. 结算与评价：平台自动扣除手续费，将剩余金额结算至承接者账户；双方完成互评，信誉分更新。

## 5.2 手续费调整流程

1. 管理员登录后台，进入“手续费配置”页面。

2. 设置手续费比例（如5%）或阶梯收费规则（如100元以下5%，100元以上3%）。

3. 点击“保存”，配置立即生效，后续所有新完成的任务均按新规则收取手续费。
