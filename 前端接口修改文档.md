# 校园任务接单平台 — 前端接口修改文档

> 版本：v1.1 | 日期：2026-03-10
> 本文档记录相对于《前端接口文档 v1.0》的所有变更，前端开发需按此文档修正对应调用逻辑。

---

## 一、全局约定变更

### 1.1 押金字段语义修正（重要）

**变更前（v1.0）**：前端接口文档将「积分」对应到 `balance`（可提现余额）字段

**变更后**：押金机制改为使用 `creditScore`（信誉分）字段

| 字段 | 含义 | 变化 |
|------|------|------|
| `balance` | 账户可提现余额，只在结算后增加 | **不再用于押金** |
| `creditScore` | 信誉分，兼作抢单押金 | 抢单时-1（冻结），结算/拒绝/超时返还+1，违规不返还 |

**前端影响**：
- `BalanceVO.frozenAmount` 字段含义改为「冻结中的信誉分」，类型为 `number`（整数）
- `UserProfileVO.creditScore` 展示时需提示「≥1才可抢单」
- 抢单前可前端预校验：`if (user.creditScore < 1) 提示「信誉分不足，无法抢单」`

## 二、任务模块变更

### 2.1 抢单接口返回值变更

**接口**：`POST /api/tasks/{id}/grab`

**变更前**：返回 `data: null`（R\<Void\>）

**变更后**：返回 `GrabVO`，前端可直接用于展示倒计时

```typescript
// 变更后的返回 data 类型
interface GrabVO {
  lockExpireAt: number  // 锁单过期时间戳（毫秒）
  message: string       // 「抢单成功！请等待发布者确认，剩余20分钟」
}
```

**前端处理**：
```typescript
const res = await taskApi.grab(taskId)
// 直接使用 res.lockExpireAt 启动倒计时，无需重新请求详情
countdownRef.value = res.lockExpireAt
```

---

### 2.2 发布者确认抢单接口返回值变更

**接口**：`POST /api/tasks/{id}/grab/confirm`

**变更前**：返回 `data: null`（R\<Void\>）

**变更后**：返回 `GrabConfirmVO`

```typescript
interface GrabConfirmVO {
  payUrl: string  // 支付页路由路径，如 "/payment/pay?taskId=xxx"
  message: string
}
```

**前端处理**：
```typescript
const res = await taskApi.confirm(taskId)
// 确认成功后直接跳转支付页
router.push(res.payUrl)
```

---

### 2.3 验收接口返回值变更（重要）

**接口**：`POST /api/tasks/{id}/verify`

**变更前**：返回 `data: null`（R\<Void\>）

**变更后**：返回 `VerifyVO`，验收通过时包含结算明细

```typescript
interface VerifyVO {
  message: string
  settlement?: {         // 验收通过时有值，验收不通过时为 undefined/null
    taskAmount: number
    feeDiscount: number  // 接单者等级折扣率，如 0.90
    feeAmount: number    // 实际手续费
    realAmount: number   // 接单者实际到账
    expGained: number    // 接单者获得经验值
  }
}
```

**前端处理**：
```typescript
const res = await taskApi.verify(taskId, { pass: true })
if (res.settlement) {
  // 展示结算明细弹窗
  showSettlementDialog(res.settlement)
}
```

---

### 2.4 任务状态流转修正（重要）

**v1.0 文档描述有误**：`grab/confirm` 后直接进入「进行中(4)」

**实际流程（已修正）**：
```
已抢单待协商(3)
  → 发布者确认  → 待支付(1)  → 发布者调用 /api/payment/pay → 进行中(4)
  → 发布者拒绝  → 待接单(2)  （信誉分返还，不记违规）
  → 抢单者取消  → 待接单(2)  （信誉分扣除，记1次违规，扣20经验）
  → 20分钟超时  → 待接单(2)  （信誉分扣除，记1次违规，扣20经验）
```

**前端影响**：
- 任务状态流转组件中，状态3之后不能直接跳4，必须经过1（待支付）
- 发布者确认后自动跳转支付页（使用 `GrabConfirmVO.payUrl`）
- 支付完成后任务变为「进行中」，不是确认完成后立即变

## 三、评价模块变更

### 3.1 查看用户评价接口返回类型变更

**接口**：`GET /api/review/user/{userId}`

**变更前**：返回裸 `Review` 实体（字段不完整，无 `taskTitle`、无嵌套 `reviewer` 对象）

**变更后**：返回 `PageResult<ReviewItemVO>`

```typescript
// 变更后（与 v1.0 文档定义一致，但之前后端未实现）
interface ReviewItemVO {
  id: string
  taskId: string
  taskTitle: string        // 新增：关联任务标题
  reviewer: {
    id: string
    nickname: string
    avatar: string
  }
  score: number
  content: string
  type: 0 | 1              // 0=发布者评接单者 1=接单者评发布者
  createdAt: number        // 毫秒时间戳
}
```

**前端影响**：前端可直接展示 `taskTitle` 和评价人头像，无需额外请求。

---

### 3.2 查看任务双方评价返回类型变更

**接口**：`GET /api/review/task/{taskId}`

**变更前**：`publisherReview`/`acceptorReview` 为裸 `Review` 实体或空 `{}`

**变更后**：两个字段均为 `ReviewItemVO` 或空 `{}`，字段结构同 3.1

## 四、支付结算模块变更

### 4.1 我的支付记录返回类型变更

**接口**：`GET /api/payment/records`

**变更前**：返回裸 `Payment` 实体（无 `taskTitle`）

**变更后**：返回 `PageResult<PaymentRecordVO>`

```typescript
interface PaymentRecordVO {
  id: string
  taskId: string
  taskTitle: string        // 新增
  amount: number
  payType: 0 | 1
  payStatus: 0 | 1 | 2
  tradeNo: string
  createdAt: number        // 毫秒时间戳（变更前为 LocalDateTime 字符串）
}
```

---

### 4.2 我的结算记录返回类型变更

**接口**：`GET /api/settlement/records`

**变更前**：返回裸 `Settlement` 实体（无 `taskTitle`，`settledAt` 为 LocalDateTime 字符串）

**变更后**：返回 `PageResult<SettlementRecordVO>`

```typescript
interface SettlementRecordVO {
  id: string
  taskId: string

  taskAmount: number
  baseFeeRate: number
  levelAtSettle: number     // 结算时等级快照
  feeDiscount: number       // 等级折扣率快照
  feeRate: number
  feeAmount: number
  realAmount: number
  expGained: number
  status: 0 | 1
  settledAt: number         // 毫秒时间戳（变更前为 LocalDateTime 字符串）
}
```

### 4.3 新增提现接口

**接口**：`POST /api/settlement/withdraw`（v1.0 已定义但后端未实现，现已补充）

请求体和返回格式与 v1.0 文档一致，无变化。

## 五、消息模块说明

消息接口返回的 `Message` 实体中时间字段 `createdAt` 为 **LocalDateTime 序列化结果**（受 Jackson 全局配置影响会输出为毫秒时间戳），前端按毫秒时间戳处理即可，与 v1.0 文档约定一致。

| `nickname` | `nickname` | 一致 |
| `role` | `role` | 一致 |
| `level` | `level` | 一致 |
| `creditScore` | `creditScore` | 一致 |
| `balance` | `balance` | 一致 |
| `totalEarned` | `totalEarned` | 一致 |
| `status` | `status` | 一致 |
| `createdAt` | `createdAt`（毫秒时间戳） | 一致 |
| — | `exp`、`bio`、`avatar`、`skills` | 额外字段，前端可忽略或展示 |

### 6.2 手续费配置接口说明

**接口**：`GET /api/admin/fee-config` / `PUT /api/admin/fee-config`

实际返回的是 `FeeConfig` 实体列表，字段与 v1.0 文档 `FeeConfigItem` 对比：

| v1.0 文档字段 | 实际字段 | 说明 |
|---|---|---|
| `id` (string) | `id` (number) | ⚠️ 注意：fee_config 使用 AUTO_INCREMENT，id 为普通 Long，**不是**雪花ID，前端应当作 `number` 处理 |
| `minAmount` | `minAmount` | 一致 |
| `maxAmount` | `maxAmount`（null 表示无上限） | 一致 |
| `feeRate` | `feeRate` | 一致 |
| `isActive` | `isActive` | 一致 |
| — | `updatedBy`、`updatedAt` | 额外字段 |

**`PUT /api/admin/fee-config` 请求体**：直接传 `FeeConfig[]` 数组，每项包含上述字段（`id` 可不传，后端会清空重建）。

### 6.3 全平台交易记录接口返回类型

**接口**：`GET /api/admin/trades`

实际返回裸 `Settlement` 实体分页，**无 `taskTitle`、`acceptorNickname`** 字段。

前端暂时用 `taskId` 展示，或自行二次请求（演示阶段可接受）。

后续如需完整字段可联系后端补充 VO。

## 七、时间字段统一说明

所有后端返回的时间字段均已通过 Jackson 全局配置序列化为**毫秒时间戳（number）**，与 v1.0 文档约定一致。

唯一例外：若某实体字段使用了 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")` 注解，则会以字符串形式返回。当前代码库中未使用该注解，故所有时间均为毫秒时间戳，前端统一用 `formatTime(ts)` 处理。

## 九、变更汇总对照表

| 接口 | v1.0 返回 data 类型 | v1.1 实际返回 data 类型 | 影响 |
|---|---|---|---|
| `POST /api/tasks/{id}/grab` | `null` | `GrabVO` | 直接获得 lockExpireAt |
| `POST /api/tasks/{id}/grab/confirm` | `null` | `GrabConfirmVO` | 直接获得 payUrl |
| `POST /api/tasks/{id}/verify` | `null` | `VerifyVO` | 验收通过含结算明细 |

## 十、任务分类标签系统（新增）

### 背景

任务分类从原来的硬编码枚举（1-5固定值）改为**动态标签系统**，管理员可在后台灵活增删改分类，前端从接口获取标签列表并渲染。

---

### 10.1 获取分类标签列表（公开接口，无需Token）

```
GET /api/categories
```

**返回 data**：

```typescript
interface TaskCategoryVO {
  id: number       // 分类ID（前端筛选时传此值）
  name: string     // 分类名称，如「代取快递」
  icon: string     // 图标，如「📦」（可直接渲染）
  sort: number     // 排序权重
  active: boolean  // 是否启用
}
```

**前端使用示例**：

```typescript
// stores/category.ts
const categories = ref<TaskCategoryVO[]>([])
async function loadCategories() {
  categories.value = await categoryApi.list()
}

// 首页标签栏渲染
// 点击某个标签 → 设置 queryParams.category = item.id → 重新请求任务列表
```

**首页标签栏交互逻辑**：

```
「全部」（不传category） + 各分类标签（传category=id）
点击标签 → router.push({ query: { category: id } })
或直接修改响应式查询参数触发列表刷新
```

---

### 10.2 任务列表接口 category 参数变更

**接口**：`GET /api/tasks`

**变更前**：`category` 为固定枚举值 1-5

**变更后**：`category` 为动态分类ID（对应 `task_category.id`），值不再限于 1-5

```typescript
// 变更后的查询参数
interface TaskQueryDTO {
  page: number
  pageSize: number
  keyword?: string
  category?: number    // 分类ID，从 GET /api/categories 获取
  minAmount?: number
  maxAmount?: number
  deliveryType?: 0 | 1
  sortBy?: 'newest' | 'amount_asc' | 'amount_desc'
}
```

---

### 10.3 发布任务接口 category 校验变更

**接口**：`POST /api/tasks`

**变更前**：`category` 必须为 1-5 的整数

**变更后**：`category` 为任意正整数（对应动态分类ID），前端发布任务时从分类列表选择

```typescript
// 发布任务表单中分类选择
const categoryOptions = await categoryApi.list()  // 从接口获取
// 用户选择后将 category.id 传入 TaskPublishDTO.category
```

---

### 10.4 管理员分类标签管理接口（新增）

所有接口需要 `role=1`（管理员Token）。

#### 获取所有分类（含禁用）

```
GET /api/admin/categories
```

返回同 `TaskCategoryVO[]`，但包含 `active=false` 的禁用分类。

#### 新增分类

```
POST /api/admin/categories
```

```typescript
interface AddCategoryDTO {
  name: string   // 分类名称（唯一）
  icon?: string  // 图标，默认「🔧」
  sort?: number  // 排序权重，默认99
}
```

返回 `data: null`

#### 修改分类

```
PUT /api/admin/categories/{id}
```

```typescript
interface UpdateCategoryDTO {
  name?: string
  icon?: string
  sort?: number
  isActive?: 0 | 1  // 0禁用 1启用
}
```

返回 `data: null`

#### 删除分类

```
DELETE /api/admin/categories/{id}
```

返回 `data: null`

**注意**：删除分类后，已发布使用该分类的任务 `categoryName` 会显示为「其他」，建议管理员仅禁用而非删除。

### 10.5 前端实现建议

**首页标签栏组件**：

```vue
<template>
  <div class="category-tabs">
    <span
      :class="{ active: !selectedCategory }"
      @click="selectCategory(null)"
    >全部</span>
    <span
      v-for="cat in categories"
      :key="cat.id"
      :class="{ active: selectedCategory === cat.id }"
      @click="selectCategory(cat.id)"
    >
      {{ cat.icon }} {{ cat.name }}
    </span>
  </div>
</template>

<script setup lang="ts">
const selectedCategory = ref<number | null>(null)

function selectCategory(id: number | null) {
  selectedCategory.value = id
  // 重新请求任务列表
  fetchTasks({ category: id ?? undefined })
}
</script>
```

**管理员后台分类配置页**（挂在 `FeeConfig.vue` 同级或合并进配置页）：
- 表格展示所有分类（含禁用）
- 支持新增、编辑名称/图标/排序、启用/禁用、删除
- 操作后调用对应管理员接口
